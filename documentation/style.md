# Coding Style

We use case to distinguish between different types of identifiers. For example, we use camelCase 
for variables and functions, and PascalCase for classes and interfaces. We also use snake_case 
for constants and private variables.

When possible, we prefer to use descriptive names for variables and functions. This helps to make the code
easier to read and understand. For example, instead of using an enumeration name like `lt`, we use `Less` or `LessThan`.
We also follow the principle of "Don't Repeat Yourself" (DRY) to avoid code duplication. This means that we try to write reusable code and avoid copying and pasting code snippets.
Finally, we use comments to explain the purpose of complex code blocks or to provide additional context. However, we avoid over-commenting and strive to write self-explanatory code whenever possible.
